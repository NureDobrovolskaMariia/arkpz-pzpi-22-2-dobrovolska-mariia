VISION AND SCOPE DOCUMENT (КОНЦЕПЦІЯ ТА РАМКИ ПРОЄКТУ)

«TechNest» - Програмна система для інкубування яєць


1 BUSINESS REQUIREMENTS (БІЗНЕС-ВИМОГИ)

1.1 Background (Передумови)

Проект «TechNest» розробляється для автоматизації та оптимізації процесу інкубації пташиних яєць. 
Інкубація є критичним етапом у виробництві птиці, оскільки вимагає суворого контролю за умовами середовища, такими як температура та вологість. 
Цей процес потребує значних зусиль для підтримання стабільних умов, що впливає на успішність інкубації та якість майбутнього молодняка.
Основна мета проєкту – створити систему, яка надасть фермерам і агропідприємствам можливість автоматизовано контролювати параметри інкубації, оперативно отримувати сповіщення про будь-які зміни умов та здійснювати аналітику для покращення процесу інкубації. 
Система допоможе зменшити ризики пов'язані з людським фактором і підвищити продуктивність за рахунок зниження витрат на постійний ручний контроль. Також починаючий фермер зможе наблюдати за процессом у режимі реального часу. 

1.2 Business Opportunities (Бізнес-можливості)

Аналіз ринку показує, що наявні рішення для автоматизації інкубаційних процесів часто мають обмежену функціональність, не забезпечують достатнього рівня аналітики або є занадто дорогими для широкого впровадження. 
Багато систем не підтримують моніторинг у реальному часі або мають складні інтеграції з іншими інструментами управління фермерськими господарствами. 
Проєкт «TechNest» ставить за мету вирішити ці проблеми шляхом впровадження інноваційного рішення з використанням IoT-технологій, сучасних веб-інтерфейсів та зручної архітектури.
Конкретні конкуренти та програмні продукти з подібним функціоналом
1.	Intelligent Hatchery Systems (IHS) - ця система відома своїми рішеннями для автоматизації інкубації яєць. Вона пропонує функції контролю за температурою і вологістю, але має обмежені можливості для аналітики та прогнозування. Вартість впровадження та обслуговування є значною, що робить її недоступною для багатьох малих фермерських господарств. Крім того, IHS не підтримує сучасні веб-інтерфейси та мобільні додатки для моніторингу в реальному часі, що зменшує її гнучкість.
2.	Petersime Hatchery Management System (PHS) - відомий бренд, що спеціалізується на розробці обладнання для інкубації та відповідного програмного забезпечення. Програма надає інструменти для контролю основних параметрів інкубації та базової аналітики. Недоліком є висока вартість обладнання та обмежена інтеграція із зовнішніми системами управління фермою. PHS орієнтований переважно на великі птахофабрики, через що невеликі підприємства можуть мати труднощі з адаптацією.
3.	ChickMaster Hatchery Management Software - система для управління інкубаційними процесами, яка фокусується на моніторингу умов і забезпечує базовий аналіз даних. ChickMaster підтримує управління параметрами температури, вологості та вентиляції. Проте її функціонал у частині аналітики та прогнозування є обмеженим. Крім того, система більше орієнтована на внутрішні процеси без можливості гнучкої інтеграції з іншими IoT-рішеннями та віддаленого доступу через мобільні додатки.
4.	Munters Hatchery Management System - спеціалізується на клімат-контролі в різних сферах, включаючи інкубаційні процеси. Їхні системи дозволяють регулювати мікроклімат, проте інтеграція з IoT-сенсорами, що можуть бути використані для збору даних в реальному часі, є обмеженою. Аналітичні можливості не є центральною функцією цієї системи, що зменшує її конкурентоспроможність для фермерів, які хочуть вивчати тенденції і робити прогнозування.
5.	Automated Egg Incubator Solutions від різних виробників - безліч автоматизованих інкубаторів, що забезпечують підтримку базових умов для інкубації (наприклад, управління температурою і вологістю). Вони часто пропонують вбудовані дисплеї для локального моніторингу, але не мають підтримки зручного веб-інтерфейсу або мобільного додатка для дистанційного моніторингу та аналітики. Такі системи можуть бути достатніми для невеликих господарств, але не забезпечують широких можливостей масштабування та аналізу історичних даних.
Основні обмеження існуючих рішень:
- 	Обмежені аналітичні можливості, зазвичай наявні рішення не забезпечують детальної аналітики та прогнозування, що важливо для підвищення ефективності процесу інкубації;
- 	Висока вартість, багато рішень мають значні витрати на налаштування та підтримку, що робить їх менш привабливими для невеликих фермерських господарств;
- 	Обмежений доступ до даних у реальному часі, не всі системи забезпечують віддалений моніторинг через мобільні додатки або веб-інтерфейси;
- 	Інтеграція з іншими системами, більшість рішень не підтримують інтеграцію з іншими IoT-пристроями або зовнішніми системами управління фермою.
Конкурентні переваги проєкту:
1.	Комплексність, система забезпечує повний цикл моніторингу та управління умовами інкубації, включаючи збір даних, їх аналіз та надсилання сповіщень у разі змін параметрів. Вона поєднує функції, що дозволяють автоматизувати більшу частину процесу та зменшити вплив людського фактора.
2.	Доступність, проєкт передбачає легку інтеграцію як для малих фермерських господарств, так і для великих агропромислових комплексів. Веб-інтерфейс та мобільний додаток забезпечать зручний доступ до системи навіть для користувачів без спеціальних технічних знань.
3.	Сучасні технології, використання IoT-сенсорів, які забезпечують точний і безперервний збір даних про параметри середовища. Завдяки цьому забезпечується реальний моніторинг із миттєвими сповіщеннями про зміну умов. Аналітичні інструменти допоможуть фермерам аналізувати історичні дані, виявляти тенденції та оптимізувати умови для максимально ефективної інкубації.
4.	Масштабованість, система розроблена з урахуванням можливості масштабування, що дозволяє збільшувати кількість підключених датчиків та інтегрувати нові функції без необхідності повного переобладнання інфраструктури.
Модель монетизації - для забезпечення сталого розвитку та доходу проєкт пропонує кілька варіантів монетизації:
- 	Ліцензійна підписка: Щомісячна або річна підписка на використання програмного забезпечення, що включає доступ до базових функцій, регулярних оновлень та базової підтримки. Користувачі можуть обрати стандартну або преміум-підписку, яка включає розширені можливості аналітики та пріоритетну технічну підтримку;
- 	Продаж обладнання: Фермери та підприємства можуть придбати необхідні IoT-сенсори для інтеграції у свою систему. Таке рішення дозволить масштабувати систему з урахуванням індивідуальних потреб кожного господарства;
- 	Додаткові сервіси для великих клієнтів: Налаштування системи під специфічні вимоги підприємств, додаткові функції аналітики та кастомізація, що спрямована на оптимізацію бізнес-процесів.
Цільовий ринок: основними клієнтами проєкту є середні фермерські господарства та великі агропідприємства, які прагнуть підвищити ефективність своїх виробничих процесів. Система також буде цікавою для спеціалізованих птахофабрик, що займаються інкубацією яєць у великих масштабах. Орієнтація на глобальний ринок можлива завдяки підтримці кількох мов інтерфейсу та використанню сучасних інтернет-технологій, що робить систему легкою для адаптації у будь-якій країні.
Перспективи розвитку: з розвитком проєкту планується додавання нових функцій, таких як автоматичне регулювання параметрів інкубації на основі аналізу даних, впровадження штучного інтелекту для прогнозування та аналізу даних, що сприятиме подальшому підвищенню якості та ефективності процесів. Можливе створення партнерських відносин із постачальниками обладнання для розширення ринку збуту та залучення нових користувачів.


2 VISION OF THE SOLUTION (КОНЦЕПЦІЯ РІШЕННЯ)

2.1 Vision Statement (Окреслення концепції)

Система для інкубування яєць розробляється як комплексне програмно-апаратне рішення, що об’єднує можливості сучасних IoT-технологій, аналітики та управління даними. Основна ідея продукту полягає в створенні ефективного інструменту для фермерських господарств і агропідприємств, який допоможе автоматизувати та покращити процес інкубації, забезпечуючи стабільний контроль за умовами інкубації яєць і їх оптимізацію.
Призначення системи, головне завдання системи – забезпечити повний цикл управління умовами інкубації, що включає збір даних у реальному часі, їх обробку, збереження та аналіз. Це дозволяє фермерам не тільки спостерігати за основними показниками (температурою, вологістю тощо), а й отримувати оперативні сповіщення у разі відхилень від заданих норм. Таким чином, фермери можуть швидко реагувати на потенційні проблеми та коригувати умови для забезпечення оптимального середовища.
Очікувані результати використання системи:
- 	Підвищення ефективності інкубації, забезпечення постійного контролю за умовами зберігання яєць та своєчасне реагування на зміни допомагає зменшити втрати і підвищити відсоток успішного виведення пташенят;
- 	Зниження людського фактора, завдяки автоматизації процесів фермери можуть уникнути помилок, пов’язаних з ручним контролем умов інкубації;
- 	Покращення якості даних, накопичення та аналіз даних дозволяють виявляти найкращі практики для підтримання стабільних умов інкубації;
- 	Економія часу і ресурсів, система мінімізує необхідність постійної фізичної присутності оператора, що дозволяє зосередитися на інших аспектах управління господарством.
Загалом, система для інкубування яєць сприяє автоматизації процесів, покращенню управління і прийняттю обґрунтованих рішень для досягнення стабільних результатів та підвищення продуктивності.

2.2 Major Features (Головна функціональність)

1.	MF-1: Автоматизований збір даних про температуру та вологість через IoT-сенсори.
Система використовує інтегровані IoT-сенсори для автоматичного збору даних про температуру і вологість у режимі реального часу. Сенсори встановлюються в інкубаційних камерах і постійно передають інформацію на сервер для моніторингу та аналізу.
2.	MF-2: Обробка зібраних даних і їх збереження у базі даних для подальшого аналізу.
Система обробляє дані, що надходять від сенсорів, фільтруючи та форматуючи їх для збереження в надійній базі даних. Це дозволяє швидко отримувати доступ до інформації та готувати її для подальшого аналізу.
3.	MF-3: Система сповіщень у реальному часі при перевищенні заданих порогових значень температури або вологості.
Система автоматично надсилає сповіщення користувачам у разі, якщо параметри температури або вологості виходять за встановлені межі. Це допомагає швидко реагувати на зміни та запобігати негативним наслідкам.
4.	MF-4: Інтерфейс для перегляду даних у реальному часі та перегляду історії умов інкубації.
Веб-інтерфейс і мобільний додаток надають користувачам можливість переглядати актуальні дані про стан інкубаційної камери, а також історію зміни параметрів за певний період. Інтерфейс є інтуїтивно зрозумілим та адаптованим для користувачів різного рівня підготовки.
5.	MF-5: Інструменти для аналізу даних, що допоможуть оптимізувати налаштування системи.
Система містить аналітичний модуль, який дозволяє проводити аналіз історичних даних для виявлення закономірностей та тенденцій у процесі інкубації. Це допомагає оптимізувати параметри інкубації, підвищуючи ефективність виведення пташенят.
6.	MF-6:  Перглянд спсоку равликів обраних за критеріями, або загалом.
Система надає можливість користувачам переглядати список равликів із застосуванням заданих критеріїв фільтрації (наприклад, вік, вид, стан здоров’я тощо) або переглядати повний список без фільтрів. Ця функція дозволяє швидко отримувати необхідну інформацію про доступні равлики, спрощуючи управління даними.
7.	MF-7:  Реєстрацію для окремих лиць. Наприклад фізичних та юредичних, задля продажу продукту.
Система забезпечує процес реєстрації користувачів, розподіляючи їх на фізичних і юридичних осіб. Це дозволяє створювати персоналізовані облікові записи для кожної категорії, забезпечуючи доступ до специфічних функцій, таких як управління замовленнями, комунікація з покупцями та оформлення документів для продажу продукції.
8.	MF-8:  Вибір пунктів фільтрації.
Система пропонує інтерактивний модуль для вибору параметрів фільтрації даних. Користувачі можуть обирати критерії, такі як розмір, вид, умови утримання чи інші характеристики, які використовуються для звуження вибірки даних. Це дозволяє швидко знаходити необхідну інформацію та приймати обґрунтовані рішення.
9.	MF-9:  Додавання, видалення, оновлення інформації.
Система забезпечує можливість керування даними: користувачі можуть додавати нові записи, видаляти зайві або застарілі дані, а також оновлювати існуючу інформацію. Ця функція гарантує актуальність бази даних та дозволяє ефективно управляти інформацією, пов’язаною з процесами обліку та управління.
Конкретні вимоги користувача та функціональні вимоги:
- 	Користувачі повинні мати можливість переглядати дані в реальному часі та отримувати сповіщення про зміни умов без затримок;
- 	Інтерфейс повинен бути зручним і зрозумілим для користувачів з різним рівнем технічної підготовки;
- 	Система повинна підтримувати масштабування для розширення можливостей та інтеграції додаткових сенсорів без втрати продуктивності.
- 	Система повинна забезпечувати надійне збереження даних і захист від несанкціонованого доступу.


3 SCOPE AND LIMITATIONS (РАМКИ ТА ОБМЕЖЕННЯ ПРОЄКТУ)

3.1 Scope of Initial Release (Рамки первинного випуску)

У початковому випуску системи для інкубування яєць будуть реалізовані базові функції, необхідні для забезпечення ефективного моніторингу та управління процесом інкубації. Першочерговий акцент зроблено на основних компонентах, які забезпечують швидку й надійну інтеграцію, простоту використання та захист даних. Цей підхід допоможе створити продукт, що відповідає вимогам різних категорій клієнтів та забезпечить їм значну цінність без високих витрат на розвиток.
Основні компоненти початкового випуску:
1.	Backend.
Реалізація API для отримання та обробки даних з IoT-пристроїв. Backend буде містити RESTful API для обробки даних, які надходять з IoT-сенсорів. Це забезпечить ефективну передачу даних, можливість інтеграції з іншими системами та швидкий доступ до інформації.
Зберігання даних у базі даних. Дані, зібрані з сенсорів, будуть зберігатися у надійній реляційній базі даних. MongoDB обрано за її стабільність, масштабованість та можливість роботи з великими обсягами даних. Це забезпечить швидкий доступ до інформації, збереження історії показників і можливість проведення аналітики.
2.	IoT.
Інтеграція датчиків температури та вологості. Датчики будуть інтегровані для постійного моніторингу ключових параметрів інкубаційних камер. Зібрані дані будуть передаватися на сервер за допомогою протоколу I²C, що забезпечує надійну і швидку передачу інформації.
Програмування мікроконтролерів. Мікроконтролери будуть запрограмовані для зчитування даних з датчиків та передачі їх на сервер. Це забезпечить стабільну роботу системи та можливість легкої адаптації до змін у вимогах.
Основні функції та переваги для клієнтів:
1.	Моніторинг у реальному часі, завдяки безперервному збору даних користувачі отримують доступ до актуальної інформації про стан інкубаційних камер, що допомагає швидко реагувати на будь-які зміни та знижувати ризик втрат.
2.	Зручний інтерфейс адміністрування, адміністративний модуль дозволяє легко налаштовувати систему, що є перевагою для невеликих фермерських господарств, які не мають ІТ-фахівців.
3.	Масштабованість та легка інтеграція, система спроектована таким чином, щоб забезпечувати можливість додавання нових датчиків і підключення до інших пристроїв без складних налаштувань.
4.	Надійність та безпека, використання протоколів шифрування для збереження даних і контроль доступу до бази даних забезпечують захист інформації від несанкціонованого доступу.
Характеристики якості:
- 	Надійність, система розроблена для безперебійної роботи 24/7, що забезпечує підтримання стабільних умов інкубації та запобігає втратам через технічні збої;
- 	Продуктивність, оптимізована архітектура гарантує швидке оброблення та збереження великих обсягів даних, забезпечуючи ефективний моніторинг у реальному часі;
- 	Доступність, інтерфейси будуть доступні через веб-браузер і мобільні пристрої, що дозволить фермерам і адміністраторам легко підключатися до системи з будь-якого місця;
- 	Простота використання, інтуїтивно зрозумілий дизайн забезпечує швидке навчання користувачів та легкість управління системою без спеціальних технічних знань.
Це базове функціональне ядро початкового випуску забезпечить суттєві переваги клієнтам, дозволяючи їм підвищити ефективність і стабільність процесів інкубації за мінімальних витрат на інтеграцію та підтримку.


4 BUSINESS CONTEXT (БІЗНЕС-КОНТЕКСТ)

4.3 Operating Environment (Робоче середовище)

Для реалізації системи обрано сучасні платформи, мови програмування, фреймворки та бібліотеки, які забезпечують стабільність, масштабованість і продуктивність. Ці технології дозволяють ефективно розробляти рішення, інтегрувати IoT-пристрої та забезпечувати високу якість програмного забезпечення.
Мови програмування:
- 	JavaScript - універсальна мова для розробки серверної логіки та API через Node.js. Підтримує асинхронну обробку запитів, що забезпечує високу продуктивність у реальному часі. Велика екосистема бібліотек через npm спрощує інтеграцію з базами даних, автентифікацією та іншими сервісами.;
- 	C/C++ - Використовується для програмування мікроконтролерів IoT-пристроїв. Забезпечує низькорівневий доступ до апаратного забезпечення, що дозволяє оптимізувати використання ресурсів і швидкість передачі даних.
Фреймворки та бібліотеки:
- 	Redux - бібліотека для керування станом у JavaScript-додатках. Вона часто використовується разом із React, але її можна інтегрувати і в інші фреймворки або проєкти;
- 	I²C - Легкий протокол обміну повідомленнями, спеціально розроблений для IoT-систем. Забезпечує швидку і надійну передачу даних між IoT-пристроями та сервером. Оптимізований для роботи в умовах низької пропускної здатності мережі;
- 	Бібліотека Wire, стандартна бібліотека Arduino, яка спрощує використання I²C-протоколу;
- 	Entity Framework Core (або SQLAlchemy) - ORM (Object-Relational Mapping), що використовується для зручної взаємодії з базою даних. Автоматизує управління даними, забезпечуючи їх цілісність та зменшуючи кількість ручного коду для роботи з SQL-запитами.
СУБД MongoDB.:
- 	Реляційна база даних із високою продуктивністю, яка забезпечує зберігання великих обсягів даних;
- 	Підтримує складні аналітичні запити, що важливо для збереження історичних даних про інкубацію та проведення аналітики;
- 	Відрізняється високим рівнем безпеки, включаючи шифрування та контроль доступу.
Основні вимоги до системи.
1. Доступність:
- 	Безперебійна робота, система має працювати 24/7 з мінімальними простоями. Для цього використовується надійна архітектура серверів і механізми автоматичного перезавантаження у разі збою;
- 	Відновлення після збоїв, реалізовано механізми створення резервних копій даних із можливістю швидкого відновлення у разі аварійного збою.
2. Надійність:
- 	Резервне копіювання, автоматичне створення резервних копій бази даних для запобігання втраті інформації. Дані зберігаються на окремих серверах для підвищення захисту;
- 	Захист від втрати інформації, використання транзакцій у базі даних гарантує, що всі операції над даними виконуються або повністю, або не виконуються взагалі.
3. Продуктивність:
- 	Швидкість обробки даних, використання MQTT-протоколу для передачі даних забезпечує мінімальні затримки у передачі повідомлень. Система обробляє тисячі запитів за секунду;
- 	Оптимізоване збереження даних, із налаштуванням для обробки великих масивів даних забезпечує швидке зчитування і запис.
4. Цілісність:
- 	Дані, що надсилаються від IoT-пристроїв, з використанням протоколу I²C, для простого і ефективного обміну даними між мікроконтролерами та периферійними пристроями.;
- 	Збереження даних, база даних захищена механізмами перевірки даних, що запобігає внесенню помилкової чи некоректної інформації.
